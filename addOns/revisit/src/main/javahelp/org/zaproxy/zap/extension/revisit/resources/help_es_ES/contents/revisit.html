Revisitar

¿Alguna vez has deseado ser capaz de navegar una aplicación que probaste con ZAP en un momento específico, quizás cuando no tienes acceso a ella, o después de que haya sido modificada de alguna forma?
Si es así ¡entonces este complemento es para ti!

The add-on allows you to instruct ZAP to return content from the ZAP history between specified dates rather than forwarding the requests to the site.
This allows you to apparently browse and use a website that you no longer have access to.
It also allows you to see what it looked like at between specific times, which is useful if the application has since changed.

When returning pages from the ZAP history floating footers are injected into HTML pages to give a visual clue that these pages are generated from ZAP.
ZAP will return pages that were the result of both GET and POST requests as long as it finds ones that match.
Pages only match if all URL and POST parameters are the same, ignoring the order and any known CSRF tokens.
If no matching page can be found in the history then a custom 404 page is returned which will include links to the same page if it has been invoked with different parameters or outside of the time period specified.

This add-on adds right click menus to the Sites Tree:

Habilitar Revisitar

Cuando se selecciona, un mensaje emergente aparecerá preguntando por los tiempos iniciales y finales que se utilizarán.
El tiempo de inicio se fija de forma predeterminada al momento de la primera solicitud en la sesión.
El tiempo de finalización se fija de forma predeterminada al momento de la última solicitud en la sesión.
Si cualquiera de los elementos de tiempo (por ejemplo "día del mes") no son cambiados, entonces esos elementos se deshabilitarán.
If the dialog OK button is pressed then the revisit functionality will be enabled for the selected site, and a 'calendar' icon added to the site node in the Sites Tree.

Deshabilitar Revisitar

Cuando se selecciona, la funcionalidad de revisitar será deshabilitada para el sitio seleccionado.

API

El complemento es compatible con los siguientes puntos de conexión API:

revisit / view / revisitList devuelve la lista de sitios con revisitar habilitado
revisit / action / revisitSiteOn Activa revisitar para el sitio especificado. Las fechas deben tener el formato: "AAAA/MM/dd HH:mm:ss"
revisit / action / revisitSiteOff Deshabilita revisitar para el sitio especificado
